# 📚 Qwen RAG — PDF-чат на русском языке

Минималистичный RAG-чат на Streamlit: загружаете PDF — задаёте вопросы, получаете ответы на основе содержимого документа.

## 🔧 Требования

- Python **3.9+**
- ~4 ГБ свободной RAM (модель работает на CPU)
- Доступ в интернет для первой загрузки моделей с HuggingFace

## 🚀 Быстрый старт

### Вариант 1 — автоматически (PowerShell)

```powershell
# В папке проекта:
.\setup.ps1
# После завершения:
.venv\Scripts\Activate.ps1
streamlit run app.py
```

### Вариант 2 — вручную

```powershell
# 1. Создать виртуальное окружение
python -m venv .venv
.venv\Scripts\Activate.ps1

# 2. Поднять БД
docker compose up -d

# 3. Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt

# 4. Запустить приложение
streamlit run app.py
```

Приложение откроется в браузере по адресу: **http://localhost:8501**

## 🧩 Компоненты

### `rag_core.py` — ядро пайплайна

| Функция | Описание |
|---|---|
| `load_pdf(path)` | Извлекает текст из PDF |
| `chunk_text(text)` | Делит текст на куски по 800 символов (перекрытие 150) |
| `embed_chunks(chunks)` | Векторизует куски через `Qwen2.5` |
| `retrieve(question, ...)` | Ищет топ-3 куска по косинусному сходству |

**Модель эмбеддингов:** `Qwen/Qwen2.5-1.5B-Instruct` — LLM переиспользуется и для генерации ответов, и для векторизации за счёт снятия скрытых состояний (hidden states).

### `app.py` — интерфейс и генерация

- Веб-интерфейс на **Streamlit**
- **LLM:** `Qwen/Qwen2.5-1.5B-Instruct` (device_map="auto", bfloat16)
- Кэширует модель и эмбеддинги в `st.session_state`, чтобы не пересчитывать при каждом вопросе

## 📊 Как это работает

```
PDF → извлечение текста → разбивка на чанки → векторизация (через ту же Qwen2.5)
                                                        ↓
Вопрос пользователя ──────────────────────→ поиск похожих чанков
                                                        ↓
                                            топ-2 чанка как контекст
                                                        ↓
                                          Qwen2.5-1.5B-Instruct (LLM)
                                                        ↓
                                                   Ответ
```

## ⚠️ Важно

- Первый запуск занимает **несколько минут** — скачивается модель (~3 ГБ).
- Векторизация чанков через LLM работает **гораздо медленнее**, чем специализированные энкодеры (особенно на CPU).
- Модель автоматически использует GPU при наличии (благодаря `device_map="auto"`).
- Если PowerShell не даёт запустить скрипт, выполните: `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
